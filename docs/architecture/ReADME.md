# Architecture

This directory documents the architecture of Team Flow Tracker using
the C4 model. Each file corresponds to one level of zoom.

## C4 levels in this project

| File | C4 Level | Question it answers |
|---|---|---|
| [context.md](context.md) | Level 1 - System Context | What is the system and who uses it? |
| container.md | Level 2 - Container | What are the deployable services? |
| component.md | Level 3 - Component | What is inside each service? |

Level 4 (Code) diagrams will appear inline in service-level
documentation when relevant.

## Diagramming approach

All diagrams are written in Mermaid and render natively in GitHub.
This keeps diagrams version-controlled and always in sync with the
decisions documented alongside them.
